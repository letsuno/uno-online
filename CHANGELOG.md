# Changelog

本文件记录项目的所有重要变更。

格式基于 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)。

## [0.4.0] - 2026-05-14

### 新增
- MCP Server（`packages/mcp`）：AI 客户端可通过 MCP 工具加入游戏，支持房间管理、游戏操作、状态查询。
- 8-bit 程序化 BGM 系统：大厅音乐、房间 BGM、音乐厅面板。
- 观众席全面改版：可视化观众列表、观众丢表情与抓 UNO、积分板加入/退出按钮、多 tab 支持。
- 双牌堆 + 弃牌堆散开布局，支持横竖屏自适应。
- 抢出村规：允许在他人回合打出完全相同的牌。
- 随机座位村规。
- 牌堆耗尽自动洗牌回收弃牌堆。
- 连续超时自动开启托管模式。
- 回合结束踢出未准备玩家，踢出改为移至观战席。
- 反作弊系统：服务端作弊检测 + 客户端全屏警告 UI。
- 玩家操作菜单：移交房主、踢人、强制静音、音量调节。
- 浏览器通知：游戏开始、轮到你、游戏结束、被踢出。
- 服务器版本更新提示。
- AI 玩家标识徽章。
- 选色光波全屏效果、+4 接受时广播全屏效果。
- 新手教程引导。
- Mumble ICE 频道管理：房间自动创建/销毁语音频道。
- 语音频道加入/离开提示音。
- game over 后可返回房间继续、房间内观战席。
- 房主可进入观战席、退出房间改为转让房主。
- 页面内 modal 替代浏览器原生 confirm/alert。
- 开始界面 overlay + 登录注册页面重设计。
- Enter 键快捷打开聊天框。

### 优化
- Docker 镜像体积 730MB → 252MB（迁移 NodeNext 模块解析）。
- 游戏信息展示改善：TopBar 牌面状态、日志补全、托管节奏优化。
- 回合冷却倒计时改为独立进度条，仅限制房主开始游戏，不影响玩家投票。
- 客户端时钟校准：连接时同步服务器时间偏移量，修复客户端时钟偏差导致倒计时异常。
- 观众席弹幕、聊天布局优化。
- 玩家与观战席分列显示，MAX_PLAYERS 不再计入观战席。
- 房间页按钮文字大小统一 + 移动端布局适配。
- 房间加入/离开状态原子化 + 客户端 teardown 统一。
- Caddy 配置抽取 SPA snippet + 默认安全头与健康检查。
- 移除游戏记录/回放系统、字体选择与圆角切换等低使用率功能。

### 修复
- 修复快速模式在质疑、选色、交换等阶段未生效的问题。
- 修复选颜色阶段被抓 UNO 后无法继续选色。
- 修复变色 +4 颜色状态异常。
- 修复罚摸期间可提前跳过的问题。
- 修复托管模式下抢出、万能牌选色、+4 卡死等多个 bug。
- 修复最后一张功能牌不触发结算胜利的问题。
- 修复质疑阶段叠加 +4 时不弹选色器导致游戏卡死。
- 修复对局记录不保存、rematch 无权限检查及断线处理逻辑。
- 修复观众离开时 spectators 列表崩溃。
- 修复自己的语音说话指示器不亮。
- 修复 GitHub OAuth 代理不生效（User-Agent 头、undici request API）。
- 修复观战席同步、表情连发节流、摸牌音效、抓 UNO 日志。
- 修复 modal 关闭缺少淡出动画、音乐厅与 BGM 状态不一致。
- 修复游戏结束后房主无法踢人 + 观战者被强制拉到玩家席。
- 修复观众主动离开未同步清理列表 + 重连解散时的注册表泄漏。

## [0.3.2] - 2026-05-10

### 新增
- 房主之外的玩家在回合结算后点击“下一局”改为投票，票数过半才开始下一局；房主仍可直接开始下一局。
- 弃牌堆改为完整状态同步，并在桌面中央展示最近 8 张的堆叠效果和 `弃牌堆 (N)` 数量。
- 所有罚摸和给牌改为逐张摸牌，并增加从牌堆飞向玩家手牌区/头像小手牌区域的动画。
- 头像旁新增本局获胜回合数和积分展示；无头像时使用 emoji 作为兜底。
- 手牌增长时在头像小手牌区域右侧显示 `+X` 滚动提示。
- UNO 和抓 UNO 增加全屏特效；抓 UNO 特效会显示谁抓了谁，并同步写入游戏日志。
- 游戏内补充音效反馈：普通出牌、罚牌、被扔道具命中都会播放音效；被扔道具会按道具类型使用不同卡通命中音。
- 房间增加闲置自动解散机制，`ROOM_IDLE_TIMEOUT_MS` 可通过 `.env` 或 `docker-compose.yml` 配置。
- 房间聊天记录会随游戏事件保存，重连、观战和回放时可以恢复显示。
- 复制房间链接、粘贴房间 URL 加入、GitHub OAuth 代理配置。
- 管理后台新增游戏记录管理。

### 优化
- “摸到能出为止”村规改为逐张摸，并在牌堆下方显示已摸张数。
- 被加牌时提示文案从“罚摸 X 张”调整为“还要摸 X 张”。
- 游戏日志合并连续摸牌记录，例如 `玩家4 摸牌 x6`，并修复 +4 重复记录问题。
- 方向圆环、虚线高亮和椭圆箭头按当前出牌方向运动，并加宽箭头。
- +2/+4 触发后增加半秒停顿，让节奏更清晰。
- 宽屏布局下右侧入口面板和游戏内容宽度适配更稳定。
- GitHub 登录按钮亮度提高，操作按钮点击区域修正。
- 语音频道成员显示玩家昵称，不再显示 `player_id`。
- 语音解码器关闭后的错误降噪，减少部分玩家听不到声音的问题。
- 文本选择在游戏页被屏蔽，避免拖拽/双击误选 UI。
- 村规可出牌判断提取为共用逻辑，并改为早停判断，减少重复遍历。
- 游戏全屏特效创建和自动移除逻辑去重，后续新增特效更容易维护。
- 项目文档和仓库链接迁移到 `letsuno` 组织，并清理过期计划/规格文档。

### 修复
- 修复房间失效后仍留在大厅列表的问题；列表会清理 `playing` 但游戏状态丢失的死房间。
- 修复结算页点击“返回大厅”只前端跳转、不离开房间导致死房间的问题。
- 修复房主返回大厅/解散房间后，其它玩家没有被自动退回大厅而卡空白页的问题。
- 修复被跳过回合后仍显示上一张出牌标记的问题。
- 修复叠加罚摸后无法继续摸牌的问题。
- 修复变色 +4 不能选颜色的问题。
- 修复打别人未喊 UNO 的罚牌可以重复触发的问题。
- 修复功能牌效果展示显示到下一家而不是被触发玩家的问题。
- 修复上传头像被 emoji 覆盖的问题。
- 修复“手牌透明”村规不生效的问题，低于阈值时头像下小手牌会展示真实牌；观战 hidden 模式也尊重该规则。
- 修复观战会话失效或恢复时可能无法正确进入观战状态的问题。
- 修复观战会话恢复构建失败的问题。
- 修复叠加罚牌规则下部分剩余手牌类村规的摸牌校验问题，并补充对应测试。
- 修复语音面板部分控制台 `AudioDecoder` closed codec 报错。
- 修复管理员面板 DELETE 请求、`/admin` 重定向、主站登录态复用、用户不存在时接口返回码等问题。
- 统一前端使用后端 API 路径，不再保留旧路径兼容。

## [0.3.0] - 2026-05-07

### 新增

#### 管理后台
- 独立管理后台应用（`packages/admin`），支持用户管理、房间管理、数据统计
- 管理端 API 插件（`/admin/*`），包含 `adminOnly` 权限中间件
- 首个注册用户自动升级为管理员
- Dashboard 统计面板：总用户数、总游戏数、活跃房间数
- 用户管理：搜索、分页、角色修改
- 房间管理：查看活跃房间、强制解散

#### 用户角色系统
- 四级用户组：普通（白色/5s冷却）、会员（绿色/2s）、VIP（金色/1s）、管理员（红色/无限制）
- 角色颜色贯穿全局：游戏内昵称、聊天、房间列表、个人资料
- 基于角色的扔道具冷却时间差异化

#### 游戏体验
- 椭圆轨道高亮弧段 + 箭头指示出牌方向
- 牌堆下方回合指示器：显示当前操作玩家头像、昵称、倒计时
- 阶段感知提示文案：质疑+4、选色、交换等状态显示对应操作文字
- 摸牌动画方向修正：卡牌飞向实际摸牌玩家位置，而非固定向下
- PC 端右上角玩家列表面板：显示所有玩家状态、手牌数、在线状态
- 游戏日志可折叠/展开
- Google 风格四色头像边框

#### 托管模式增强
- 自动质疑 +4（`challenging` 阶段）
- 自动选色（选手牌最多的颜色）
- 自动选择交换对象（选手牌最多的玩家）

#### 主题系统
- 圆角/科技双风格主题，支持一键切换
- 主题感知的 CSS 变量（`--radius-btn`, `--radius-input`, `--radius-card-ui`, `--radius-panel-ui`）
- 科技风格额外边框发光效果
- 统一 Input 组件（cva 变体：default/ghost，尺寸：default/lg/sm）
- Button 组件新增 `ghost` 和 `outline` 变体

#### 构建信息
- 首页左下角显示版本号和构建日期
- 游戏 TopBar 显示版本号

#### 文档
- CLAUDE.md 项目总览
- 前端开发规范（目录结构、命名、组件、样式、状态管理）
- 后端开发规范（插件架构、路由、数据库、安全规范）
- 插件扩展规范（服务端/客户端模板、开发流程）
- 村规扩展指南（Rule 注册表模式、RuleDefinition 接口）

### 修复
- 修复 `/profile` 接口泄露 `passwordHash` 到客户端的安全漏洞
- 修复服务端测试文件 import 路径（适配插件目录结构）
- 修复 `chat:message` 字段命名：`username` 改为 `nickname`（语义一致）
- 修复质疑 +4 阶段回合指示器显示错误玩家（应显示 `pendingDrawPlayerId`）
- 修复 `choose_color` 进入 `challenging` 后计时器被停止的 bug
- 服务端超时兜底：质疑/选色/交换阶段超时自动执行默认操作，防止玩家离线卡死

### 重构
- **服务端插件架构**：Auth、Profile 迁移为 Fastify 插件；Room、Game、Voice、Interaction 模块移入 `plugins/core/` 目录
- **客户端 Feature 模块**：所有页面/组件/store 按功能域重组为 `features/`（auth、game、lobby、profile）+ `shared/`
- 创建 `PluginContext` 和 `plugin-loader` 基础设施
- 统一 `SocketData` 接口到 `ws/types.ts`
- 创建 `useEffectiveUserId` hook 消除 7 处重复的 `viewerId ?? authUserId` 模式
- 清理死代码：`shared-state.ts`、`createAuthHook`、`refreshGameStateTTL`、`cleanupRoomVoice`
- 移除不必要的 export、精简 `kv/index.ts` 重导出
- 删除废弃组件 `DirectionIndicator`、`OpponentRow`
- 删除死代码 `redis/client.ts`
